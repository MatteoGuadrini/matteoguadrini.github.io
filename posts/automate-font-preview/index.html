Automate Font Preview
devops system design font python bash design
Published: 2021-05-16

In the life of a designer or font maker, the need arises to see how a font looks on a certain image, with a certain color and size, or simply, have a reliable preview of the final result. To do this, graphical tools are required.

Preview of a font

To work with font previews, there are system previewers at the level of any operating system. This is a great starter tool to see what a font will look like on our project. The problem is that you can’t do many customizations with these tools.

Graphics tool

To change phrase, color, size and background you need advanced image tools, such as Photoshop, GIMP, etc.

But even here, we can see font previews only from the font selection menu.

To customize the sentence, for example, we have to insert the text in our project and scroll through the fonts to be tested one by one. Let’s say that we would like to test only the system fonts (at least a hundred), it takes a long time; and this, just for one project.

Command tool

A quicker way is to use command line tools. These can be faster if used wisely.

The most popular being written in bash, is font preview which comes as a font selector. As you select a font, the tool presents a quick preview of what the font will look like.

This is very useful, but to customize color and size, you have to configure it by exporting variables, see doc. It’s fine for a single preview, but for many, it gets cumbersome.

Web tool

There may also be web alternatives, but even these are limited to the fonts set in the application. Here are a few:

  • WebFontPreview
  • FontPreview
  • Google Fonts Preview tool

Python fontpreview

To get dynamic previews based on different fonts, you need a tool that does not interact with the user. The python library fontpreview, helps us to automate and speed up this process. Let’s see how.

Develop a simple previewer

First, let’s read the documentation and install the package. We understand from it that there are various objects that could do for us.

The FontPreview object is just right for us. Let’s see how to create a preview of a font in just three lines of code.

from fontpreview import FontPreview
fp = FontPreview('/tmp/noto.ttf')   # path of font file
fp.save('/tmp/fp.png')              # default directory is working directory

In this way, however, we have obtained a preview without customizations, which means that we have equaled the other tools mentioned above.

But this object allows a series of customizations that are right for us. Now let’s modify our script, so that the background color is blue while the font color is yellow and the sentence as we want it. Also, we can set a default folder /home/font_previews, so that every font we put in it will create a preview.

from fontpreview import FontPreview
import os

FONT_PATH = '/home/font_previews'

for file in os.listdir(FONT_PATH):
	font_file = os.path.join(FONT_PATH, file)
	fp = FontPreview(font_file,
                	font_size=50,
                	font_text='My personal preview',
                	bg_color='blue',
                	fg_color='yellow')
	fp.save(f'/tmp/{file}.png')

Now, we can run 100 font previews in less than two seconds. Let’s save this file as blue-yellow-font-previewer.py and try it on the command line.

$ python3 blue-yellow-font-previewer.py

More complex preview

This library not only has objects for previewing fonts, but also special objects for advanced image composition based on fonts.

Let’s imagine that some designers give us indications of color, size, background images and other information about the final project. With this library we can really range in any type of layout.

A complex and fascinating example is reproducing the preview of the Fira Code font that can be seen on the official GitHub page Fira Code preview.

Let’s see how to code this preview with the fontpreview module.

from fontpreview import FontBanner, FontWall
fira_code = '/tmp/firacode.ttf'
# RGB group = ('background', 'FIRA COD color', 'Ligature color', 'E color background')
colors_group = [
                ('black', (0, 143, 0), (0, 236, 236), (255, 0, 255)),
                ('black', (166, 47, 123), (81, 208, 93), (11, 179, 248)),
                ((13, 21, 43), (112, 204, 84), (226, 110, 34), (223, 245, 90)),
                ((43, 6, 42), (136, 126, 135), (4, 150, 153), (147, 103, 145)),
                ((39, 57, 85), (255, 241, 208), (208, 84, 0), (209, 215, 227)),
                ((31, 63, 89), (248, 248, 242), (230, 219, 117), (166, 226, 51)),
                ((1, 47, 80), (224, 202, 52), (73, 217, 38), (255, 125, 158)),
                ((0, 0, 170), (75, 224, 245), (255, 255, 85), (0, 170, 170)),
                ('white', 'black', 'black', 'black'),
                ((247, 247, 247), (167, 29, 93), (121, 93, 163), (0, 134, 179)),
                ((239, 240, 243), (15, 131, 207), (208, 84, 0), (105, 40, 122)),
                ((239, 231, 212), (218, 116, 53), (0, 142, 212), (186, 136, 0)),
                ((39, 40, 34), (132, 214, 45), (249, 39, 114), (174, 129, 255)),
                ((43, 48, 59), (180, 142, 173), (143, 161, 179), (152, 190, 140)),
                ((32, 32, 32), (171, 130, 84), (160, 171, 127), (216, 127, 98)),
                ((0, 43, 54), (0, 160, 153), (126, 143, 3), (218, 66, 130))
                ]
banners = []
# Create banners
for colors in colors_group:
    # Create a FontBanner objects
    fb = FontBanner(fira_code, (413, 240))
    liga = FontBanner(fira_code, (413, 240))
    E = FontBanner(fira_code, (40, 70))
    # Set background colors
    fb.bg_color = liga.bg_color = colors[0]
    E.bg_color = colors[3]
    # Set foreground colors
    fb.fg_color = colors[1]
    liga.fg_color = colors[2]
    E.fg_color = colors[0]
    # Set text
    fb.font_text = 'FIRA COD'
    liga.font_text = "!=->>++:="
    E.font_text = 'E'
    # Set text position
    E.set_text_position('ltop')
    fb.set_text_position((25, 60))
    liga.set_text_position('top')
    # Adding image on fb
    fb.add_image(liga, (0, 122))
    fb.add_image(E, (339, 60))
    # Add to list of banners
    banners.append(fb)

# Create a wall
fw = FontWall(banners, max_tile=4)
fw.save('/tmp/fira_code.png')

And this is result.

Command line preview

This package also comes with a simple command line tool that is used to make simple previews. Of course, if placed in the right context, such as a bash loop, it is possible to run several previews with a single script.

#!/bin/bash
for filename in $(/home/font_previews/*); do
    fp -t "My personal preview" -b "blue" -f "yellow" -z 50 -s "/tmp/${filename}.png" /home/font_previews/$filename
done

This simple script performs the same result as the python script in the “Develop a simple previewer” section.

Conclusion

There are many differences between designers and DevOps. Designers use tools that require a lot of human intervention, DevOps try to use the keyboard and mouse as little as possible. Both have creativity. Using these tools, such as fontpreview, bridges this gap that exists between the world of DevOps and designers and make everyone a little freer by saving time.
