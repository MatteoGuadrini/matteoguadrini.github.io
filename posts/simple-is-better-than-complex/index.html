Simple Is Better Than Complex
Published: 2020-12-20

…and complex is better than complicated.

This has always been my motto. All my lines of code have simplicity as their primary goal.

I decided to start this blog, to tell about my IT adventures, to describe all my projects and experiences, so that they can help others.

Open Source

To apply my motto, one of the principles that drives all my works, is the open source philosophy. Each of my projects is open source. Every single line of code is for everyone who wants to use my libraries and tools, but above all who wants to contribute.

Python

My favorite language for creating nice and useful tools, libraries and things is python.

I learned to write in python in the year 2005, in python 2.4. Initially, I wrote some scripts for client automatisms. Nothing really useful but it all starts with studying and “random things”.

In December 2008, python 3 was released. I abandoned development in python 2 and embraced the new release. With this version I started to seriously study the language; initially through Mark Lutz’s “Learning Python” book and then with the following book “Programming Python”.

Linux

My first operating system was Mandrake with KDE 1.2 as the desktop environment. Since then, Linux kernel based operating systems have accompanied me in my personal and work life. In 2008, along with python 3, fedora core (now fedora only) is my favorite operating system. I also used debian based OS, like ubuntu and other distros.

Other languages

Over the years and my studies, I have learned various programming languages. Bash was one of the first, followed by Perl and PHP for web programming. C# and powershell on Windows environments. But python, it always remains my favorite.

Let’s go!

My life is decorated with personal and work projects that I want to describe and tell you readers. This blog was born for those who want to learn or compare their works and libraries with mine. It was also born to try to describe all the use cases of my projects but above all to entice people to work with my projects and maybe … develop new features. Everyone is welcome.
